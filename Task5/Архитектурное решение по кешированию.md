MES API довольно нагруженный сервис. Во время обсчета заказа он может быть недоступен. Можно внедрить кеш для сервиса MES API, который будет хранить состояние активных заказов и активные модели.

# **Мотивация**

Внедрение серверного кеширования для приложения MES API позволить ускорить время отклика на сайте. Доступ к кешу можно дать и Web приложению MES. Пока сервис MES API будет обсчитывать заказ, данные о текущем статусе и описание можно брать из кеша. Когда сервис MES API обсчитает заказ, он может обновить данные в кеше. Изменения тоже лучше проводить в кеше, с последующей записью данных из кеша в базу.

Такой промежуточный кеш между сайтом, сервисом и базой позволит ускорить отклик для клиентов, позволит ускорить работу сервиса. 

# **Предлагаемое решение**

Реализация кеша по стратегии **Read-Through** позволит сервису MES API обновлять данные в кеше, а MES будет читать их оттуда. Прогрев кеша будет на стороне сервиса MES API. Для записи подойдет стратегия **Write-Through**. Сервис будет писать данные в кеш, а то будет обновлять данные в кеше.

[Диаграмма последовательности действий](docs/site/diagram/index.html)

В кеше будем сохранять:

- данные заказов;

- данные по операторам. 

Рассмотрим подходящие стратегии инвалидации кеша.

| Стратегия                           | Подходит                                                                                                                   | Не позволяет                                                                                                        |
| ----------------------------------- | -------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------- |
| Временная инвалидация               |                                                                                                                            | Не позволяет оперативно реагировать на изменение состояния заказа                                                   |
| Инвалидация, основанная на запросах | Можно по запросу пользователя изменить состояние заказа в кеше, что быстро забронирует его и покажет это другим операторам | Изменить заказ в кеше может сервис MES API. Это может привести к несогласованности данных.                          |
| Инвалидация на основе изменений     | Изменение статуса заказа быстро покажет это остальным операторам.                                                          |                                                                                                                     |
| Программная инвалидация             | Можно быстро обновлять данные по заказам                                                                                   |                                                                                                                     |
| Инвалидация по ключу                |                                                                                                                            | Изменениями в кеше управляет MES API, поэтому изменение со стороны сайта может привести к несогласованности даннызх |
